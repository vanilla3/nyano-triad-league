# NYTL UI/UX QA Checklist v1

目的: 「見た目が良い」だけでなく、触ったときの “気持ちよさ” と “壊れていない” を担保する。

---

## A. 表示/文言

- [ ] Match 画面で文字化けが無い（例: `繧` `縺` `繝` が出ない）
- [ ] 制御文字（U+0080）が無い
- [ ] アイコンが豆腐にならない（PUA 依存が無い）
- [ ] 日本語/英語の混在が意図通り（2言語併記は最小）

---

## B. 操作の気持ちよさ（桜井レンズ）

- [ ] タップ/クリック直後に反応がある（押下/ハイライト）
- [ ] 決定時に “確定感” がある（軽いバウンド/光/音）
- [ ] 間違えた操作がすぐに分かる（置けない場所/ターン違い）
- [ ] 置ける場所・次の行動が分かる（視線誘導/Idle guide）

---

## C. 演出（VFX / Motion）

- [ ] important moment（反転/勝敗/確定）だけ派手にする
- [ ] `prefers-reduced-motion` でアニメが抑制される
- [ ] VFX low で粒子やシェイクが減る/消える
- [ ] FOUC（CSS読み込み前に崩れる）が目立たない

---

## D. 盤面（Board）

- [ ] セルの境界が分かる（薄い縁 + 影）
- [ ] 選択中カードが視認できる（浮き/光/枠）
- [ ] 反転が分かる（色・明度・小さなVFX）
- [ ] レイアウトのジャンプが無い（Nyano コメント等）

---

## E. 共有（Shareworthy）

- [ ] URL/JSON/リプレイなど共有導線が分かる
- [ ] 共有ボタンが “押したくなる” 見た目と文言
- [ ] 共有後にユーザーが迷子にならない（戻りやすい）

---

## F. パフォーマンス/モバイル

- [ ] モバイル幅でタップ領域が小さすぎない（44px 目安）
- [ ] 背景アニメが重すぎない（FPS低下・ファンが回る等）
- [ ] 画像が巨大すぎない（tile は適切なサイズ・圧縮）

---

推奨: 重大バグ発見時は `docs/03_ops/PLAYTEST_LOG.md` に追記する。
