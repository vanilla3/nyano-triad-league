# NYTL Game UI Quality Principles v1（Sakurai Lens）

このドキュメントは Nyano Triad League の UI/UX を「ゲーム品質」に引き上げるための、
**思想（principles）とチェックリスト**です。

※ ここで言う “Sakurai Lens” は、個人の発言引用ではなく、
**「遊びやすさ」「理解しやすさ」「手触り」へ極端にこだわる視点**の便宜的な呼び名です。

---

## 1. UI は飾りではなく、プレイ体験の一部

UI は「情報を置く器」ではなく、
**プレイヤーの意思決定を誘導し、行動の結果を返す“ゲーム装置”**です。

### ゴール

- 触る前に「押せる」が分かる（affordance）
- 押した瞬間に「押せた」が返る（feedback）
- 迷った瞬間に「次にやること」が自然に目に入る（guidance）

---

## 2. 体験設計の三原則

### 2.1 一秒で理解、ゼロ秒で反応

- **1秒**：画面を見た瞬間に「今どこで、何ができて、何をすべきか」が分かる
- **0秒**：入力に対して反応が即座に返る（視覚/音/振動のいずれか最低1つ）

補足:
- 0秒は比喩です。実装上は **100ms 以内**に「押した感」が返るのが目安。
- 処理が重い場合は「先に反応、後で完了」（optimistic feedback + loading）で体験を守ります。

### 2.2 一画面一主役

どの画面も “主役のボタン” は 1つ。

- 主役は **サイズ/彩度/光/余白**で勝たせる
- サブ行動は「押せるが目立たない」に落とす
- 例外：対戦中（主役が状況で切り替わる）

### 2.3 失敗しにくいのに、自由

- 誤タップしにくい（余白/ヒット領域/危険操作の確認）
- ただし “毎回確認” は体験を壊す。**危険操作だけ**に絞る
- 上級者にはショートカット（キーボード/連打/長押し）を用意しても良い

---

## 3. “触れそうに見える” マテリアルの条件

ゲーム UI のボタンは、ただ角丸で影があれば良いわけではありません。

### 3.1 立体のレイヤー（最低3層）

1) **外縁（rim）**：背景から切り出す輪郭
2) **面（face）**：押す面。グラデーションで上面/下面の光を表現
3) **艶（specular）**：上側の薄いハイライト。ガラス/樹脂っぽさの核

加点:
- **ノイズ（grain）**：ベタ塗りの “のっぺり” を消す
- **内側影（inner shadow）**：沈み/凹み表現（盤面スロットに効く）

### 3.2 状態変化は “物理” で統一

- hover：少し浮く（影が伸びる）
- active：沈む（影が潰れる + 面が暗くなる）
- selected：縁色/発光が変わる（押せるの上位状態）
- disabled：彩度とコントラストを落とす（透明度だけに頼らない）

---

## 4. 情報設計（迷いを削る）

### 4.1 “いま必要な情報” だけ先に

例: 対戦中

- 先に：ターン/スコア/次にやること
- 後で：詳細ログ/ルール全文/デバッグ

### 4.2 人は文章を読まない。だが読める人は救う

- 第一段階：アイコン + 1語（Decks, Arena など）
- 第二段階：短文（12〜18字）
- 第三段階：詳細（説明モーダル/ヘルプ）

---

## 5. モーションと演出

### 5.1 演出は “結果の可視化”

- 置いた → 置けた
- ひっくり返った → 連鎖した
- 勝った → 勝った理由が分かる

### 5.2 速度のルール（目安）

- micro：120〜180ms（ボタン押下、hover）
- small：200〜280ms（画面内の小さな遷移）
- big：350〜550ms（勝敗/シーン転換。頻発させない）

### 5.3 `prefers-reduced-motion` と `data-vfx`

- off：アニメ停止（必要最低限の状態差のみ）
- low：短縮/回数削減
- medium/high：演出を許可

---

## 6. 画面別チェックリスト

### Home / Arena

- [ ] 主役の導線（Play/Quick Play）が “最も大きく・最も光っている”
- [ ] 触った瞬間に反応（色/影/音）が返る
- [ ] 次の一手（デッキ→対戦→共有）が視線誘導される

### Decks（図鑑/デッキビルダー）

- [ ] カードが主役。UI がカードに勝たない
- [ ] フィルタは “押せる” 形（チップ/ピル）で、現在状態が一目で分かる
- [ ] 390px 幅で操作が詰まらない（横スクロールを発生させない）

### Match（盤面）

- [ ] 空きマスが “置き場” に見える（凹み/内側影/スロット感）
- [ ] 置ける場所が迷わず分かる（ハイライトが強いが、眩しすぎない）
- [ ] テキスト表示でレイアウトが揺れない（コメント/サマリーの固定スロット）

### ルール設定

- [ ] 初見でも 2クリック以内で対戦開始できる（推奨プリセット）
- [ ] “Advanced” は折りたたみ。だが到達可能
- [ ] 選んだルールが短文で説明される（何が変わるか）

## 8. ありがちな落とし穴（先に潰す）

- 透明度だけで “無効” を表現する → 読めなくなる
- blur を多用してモバイルが重くなる → “ガラスっぽさ” は艶と縁で作る
- 画面を賑やかにしすぎて主役が消える → 主役は必ず勝たせる
