### 📚 参考文献・理論的基盤

| 書籍/理論 | 核心的洞察 |
|-----------|-----------|
| Don Norman『誰のためのデザイン?』 | アフォーダンス・シグニファイア・制約・対応付け |

## 🔬 UIの7階層モデル（独自フレームワーク）

### Layer 1: 知覚層（Perception Layer）
```
視覚処理の最下層 — 0.1秒以内の無意識的処理
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **コントラスト** | 明度差・彩度差・色相差 | WCAG 2.1 AA以上（4.5:1テキスト、3:1大テキスト） |
| **視覚階層** | サイズ・重み・位置・色による優先度 | 3秒で主要アクションを特定可能 |
| **図と地** | フォーカス対象と背景の分離 | 曖昧さゼロ、境界明確 |
| **色彩心理** | 感情喚起・文化的意味・ブランド連想 | 意図的な感情設計 |

### Layer 2: 認知層（Cognition Layer）
```
理解と解釈 — 0.5-3秒の処理
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **情報密度** | 単位面積あたりの情報量 | 認知オーバーロードを避ける最適密度 |
| **チャンキング** | 情報のグループ化単位 | 7±2法則、意味的まとまり |
| **視線誘導** | F/Z パターン、視覚的フロー | 自然な読み順で重要情報に到達 |
| **メンタルモデル整合** | ユーザーの既存概念との一致 | 学習コストゼロ |
| **アフォーダンス** | 操作可能性の視覚的暗示 | ボタンは押せる、リンクはクリック可能に見える |

### Layer 3: 操作層（Interaction Layer）
```
行動とフィードバック — ミリ秒〜秒単位
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **レスポンシブネス** | 入力→反応の遅延 | <100ms 即座、<1s 許容、>3s 離脱 |
| **フィードバック種類** | 視覚/聴覚/触覚・即時/遅延・成功/失敗 | 全操作に適切なフィードバック |
| **エラー防止** | 制約・確認・取消可能性 | ユーザーのミスを設計で防ぐ |
| **直接操作感** | ドラッグ、ピンチ、スワイプの直感性 | 物理法則に準拠した動き |
| **ターゲットサイズ** | タッチ/クリック領域の大きさ | 最小44x44px（モバイル） |

### Layer 4: 構造層（Architecture Layer）
```
情報設計と導線 — セッション全体
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **ナビゲーション明快さ** | 現在地・行き先・戻り方 | 常に「今どこ？」が分かる |
| **情報アーキテクチャ** | 分類・階層・ラベリング | ユーザーの言葉で、論理的に |
| **プログレッシブ開示** | 段階的な情報・機能の提示 | 必要な時に必要な分だけ |
| **一貫性** | パターン・用語・位置の統一 | 予測可能性の担保 |
| **フロー効率** | タスク完了までのステップ数 | 最短経路での目標達成 |

### Layer 5: 感情層（Emotional Layer）
```
体験の質 — 長期記憶に残る印象
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **美的喜び** | 視覚的美しさ、調和、独自性 | 「美しい」と感じる瞬間 |
| **マイクロデライト** | 予期せぬ小さな喜び | Easter egg、意外性のあるアニメーション |
| **信頼感** | プロフェッショナリズム、安心感 | デザイン品質＝サービス品質の暗示 |
| **パーソナライゼーション感** | 「自分のため」という感覚 | カスタマイズ可能性 |
| **達成感** | タスク完了時の満足感 | 適切な祝福・承認のフィードバック |

### Layer 6: 文脈層（Context Layer）
```
使用環境への適応
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **デバイス適応** | 画面サイズ、入力方式、性能 | 全デバイスで最適体験 |
| **環境適応** | 明るさ、ノイズ、移動中/静止 | ダークモード、大きめUI等 |
| **時間的文脈** | 急いでいる/じっくり、初回/リピート | 状況に応じたUI切り替え |
| **心理的文脈** | ユーザーの気分・目的・専門度 | アダプティブUI |

### Layer 7: 意味層（Meaning Layer）
```
ブランドと物語
```

| 要素 | 分解 | 良いUI基準 |
|------|------|-----------|
| **ブランド一貫性** | トーン、声、ビジュアル言語 | ブランドDNAの視覚化 |
| **ストーリーテリング** | ナラティブ構造、感情曲線 | 体験を物語として設計 |
| **価値提案の明示** | なぜこのサービスか | 3秒でUSPが伝わる |
| **世界観構築** | 独自の宇宙・概念体系 | 没入感、帰属意識 |

### 💎 各カテゴリの黄金パターン抽出

#### ダッシュボード系
```
Linear, Notion, Figma, Vercel
```
- 情報密度の最適化（余白の戦略的使用）
- コマンドパレット（⌘K）による高速操作
- モノクロベース + アクセントカラー1色
- 状態の視覚化（ステータスバッジ、プログレス）

#### マーケティングLP系
```
Stripe, Vercel, Linear, Raycast
```
- ヒーロー区画の圧倒的インパクト
- スクロール連動アニメーション
- 社会的証明の巧みな配置
- CTAの視覚的突出

#### クリエイティブ/ポートフォリオ系
```
Apple, Porsche, Bang & Olufsen
```
- 大胆な余白
- 製品/作品を主役にする抑制されたUI
- 映像的・没入的体験
- 細部への執着（1pxの精度）
